# MiniProject01-PrincessMaker
# 프린세스 메이커 게임 프로젝트 개요

## 1. 프로젝트 기본 정보

- **프로젝트 명**: Princess Maker (프린세스 메이커)
- **개발 언어**: Java
- **개발 환경**: IntelliJ IDEA, Console 기반 실행
- **프로젝트 설명**: 24개월 동안 프린세스를 육성하여 다양한 엔딩을 달성하는 육성 시뮬레이션 게임

---

## 2. 게임 개요

### 게임 장르

- 육성 시뮬레이션 / 텍스트 기반 어드벤처

### 게임 목표

- 프린세스를 24개월(24턴) 동안 육성하여 원하는 엔딩에 도달하기

### 플레이 방식

- 매 턴마다 3가지 활동 중 하나를 선택
- 활동에 따라 능력치가 상승
- 12턴 후 직업을 선택하여 특정 능력치에 보너스 획득
- 최종 능력치에 따라 5가지 엔딩 중 하나로 분기

### 게임 기간

- **총 24턴** (1턴 = 1개월)
- 시작: 2025년 1월
- 종료: 2026년 12월

---

## 3. 핵심 게임 시스템

### 3-1. 능력치 시스템

| 능력치 | 설명 | 초기값 |
| --- | --- | --- |
| **체력** (Physical) | 육체적 활동과 관련된 능력 | 0 |
| **지능** (Intelligence) | 학습 및 지적 활동과 관련된 능력 | 0 |
| **매력** (Charm) | 사교 및 외모와 관련된 능력 | 0 |

**증가 방식**:

- 활동 선택 시 랜덤하게 증가 (주 능력치 10~15, 부 능력치 3~5)
- 직업 보너스 적용 시 특정 능력치 50% 추가 증가

---

### 3-2. 활동 시스템

| 활동 | 주 증가 능력치 | 부 증가 능력치 | 설명 |
| --- | --- | --- | --- |
| **운동** | 체력 ↑↑ (10~15) | 지능 ↑ (3~5) | 육체적 단련 활동 |
| **공부** | 지능 ↑↑ (10~15) | 매력 ↑ (3~5) | 학습 및 지적 활동 |
| **사교** | 매력 ↑↑ (10~15) | 체력 ↑ (3~5) | 사람들과의 교류 활동 |

**활동 실행 프로세스**:

1. 3가지 활동 중 선택
2. ~~활동 진행 애니메이션 출력 (0.5초마다 "." 표시)~~
3. 결과 출력 (대성공! 능력치 상승)
4. 현재 능력치 표시

---

### 3-3. 직업 시스템

**선택 시점**: 12턴 완료 후

| 직업 | 보너스 능력치 | 보너스 효과 | 특징 |
| --- | --- | --- | --- |
| **군인** (Soldier) | 체력 | +50% | "군인답게 ~을(를) 수행했다!" |
| **학자** (Scholar) | 지능 | +50% | "학자답게 ~을(를) 수행했다!" |
| **연예인** (Celebrity) | 매력 | +50% | "연예인답게 ~을(를) 수행했다!" |

**직업 효과**:

- 보너스 능력치와 활동의 주 증가 능력치가 일치할 때 50% 추가 증가
- 예: 군인이 운동을 하면 체력이 15~22 증가 (기본 10~15의 150%)

---

### 3-4. 엔딩 시스템

| 엔딩 | 조건 | 설명 |
| --- | --- | --- |
| **장군 엔딩** | 체력 ≥ 150 | 뛰어난 체력으로 장군이 됨 |
| **교수 엔딩** | 지능 ≥ 150 | 뛰어난 지능으로 교수가 됨 |
| **스타 엔딩** | 매력 ≥ 150 | 뛰어난 매력으로 스타가 됨 |
| **평범한 삶 엔딩** | 아무 능력치 ≥ 100 | 평범하지만 행복한 삶을 살게 됨 |
| **백수 엔딩** | 모든 능력치 < 100 | 아무것도 하지 않고 백수가 됨 |

**엔딩 판정 우선순위**: 장군 → 교수 → 스타 → 평범한 삶 → 백수

## 4. 게임 진행 흐름

```
[시작]
   ↓
[이름 설정]
   ↓
[1~12턴: 활동 선택 반복]
   ├─ 활동 선택 (운동/공부/사교)
   ├─ 활동 실행
   ├─ 능력치 증가
   └─ 다음 월로 이동
   ↓
[12턴 후: 직업 선택 이벤트]
   ├─ 군인/학자/연예인 중 선택
   └─ 취직 메시지 출력
   ↓
[13~24턴: 활동 선택 반복]
   ├─ 활동 선택 (직업 보너스 적용)
   ├─ 활동 실행
   ├─ 능력치 증가
   └─ 다음 월로 이동
   ↓
[엔딩 판정]
   └─ 최종 능력치에 따라 5가지 엔딩 중 하나

```

### 주요 이벤트

1. **게임 시작**: 환영 메시지 및 이름 설정
2. **매 턴**: 날짜 표시, 활동 선택, 결과 확인
3. **12턴 후**: 직업 선택 이벤트
4. **24턴 후**: 엔딩 연출

### 5-3. 적용된 객체지향 개념

### 1. **상속 (Inheritance)**

- `Job` 추상 클래스를 `Soldier`, `Scholar`, `Celebrity`가 상속
- 공통 기능은 부모 클래스에서 구현, 직업별 특성은 자식 클래스에서 구현

```
Job (추상 클래스)
 ├── Soldier
 ├── Scholar
 └── Celebrity

```

### 2. **추상 클래스 (Abstract Class)**

- `Job` 클래스: 모든 직업의 공통 틀 제공
- 추상 메서드: `getStatBonus()`, `getActivityMessage()`
- 자식 클래스가 반드시 구현해야 함

### 3. **다형성 (Polymorphism)**

- `Princess`가 `Job` 타입으로 직업을 저장
- 실행 시점에 실제 직업(Soldier/Scholar/Celebrity)에 따라 다른 동작
- 예: `princess.getJob().getActivityMessage()` → 직업마다 다른 메시지

### 4. **캡슐화 (Encapsulation)**

- 모든 클래스의 필드를 `private`로 선언
- `getter`/`setter` 메서드로만 접근 가능
- 데이터 무결성 보호

### 5. **단일 책임 원칙 (Single Responsibility Principle)**

- 각 클래스가 하나의 명확한 역할만 수행
    - `Princess`: 데이터 저장
    - `ActivityService`: 활동 처리
    - `GameService`: 게임 흐름 제어
    - `ConsoleView`: 입출력 처리

## 6. 구현 특징

### 6-1. 수정/추가 친화적 설계

- **단순한 클래스 구조**: 복잡한 디자인 패턴 배제
- **명확한 책임 분리**: 각 클래스의 역할이 직관적
- **주석 없이 읽기 쉬운 코드**: 메서드명과 변수명만으로 이해 가능

### 6-2. 콘솔 기반 인터페이스

- **Scanner**를 통한 사용자 입력
- **System.out.println**을 통한 출력
- **진행 애니메이션**: Thread.sleep을 활용한 시각적 효과

### 6-3. 확장 가능한 구조

- **새 활동 추가 용이**: Activity 배열에 추가만 하면 됨
- **새 직업 추가 용이**: Job을 상속받는 클래스 추가
- **새 엔딩 추가 용이**: showEnding() 메서드에 조건 추가
- **능력치 추가 가능**: Princess 클래스에 필드 및 메서드 추가

### 6-4. 랜덤 요소

- **Random** 클래스 활용
- 활동 성공 시 능력치 증가량 변동
- 매 플레이마다 다른 결과 제공

### 6-5. 날짜 시스템

- **LocalDate** 활용
- "yy/MM" 형식으로 현재 날짜 표시
- 매 턴마다 자동으로 1개월 증가

## 8. 개발 가이드

### 실행 방법

1. IntelliJ IDEA에서 프로젝트 열기
2. `Main.java` 파일 우클릭
3. "Run 'Main'" 선택
4. 콘솔에서 게임 진행

### 클래스 구현 순서

1. **Princess, Activity** (기본 데이터 클래스)
2. **Job, Soldier, Scholar, Celebrity** (직업 시스템)
3. **ConsoleView** (입출력 처리)
4. **ActivityService** (활동 처리 로직)
5. **GameService** (전체 흐름 제어)
6. **Main** (실행)

### 테스트 방법

- 각 엔딩 도달 가능 여부 확인
- 직업 보너스 정상 작동 확인
- 날짜 표시 정확성 확인
- 능력치 계산 정확성 확인
